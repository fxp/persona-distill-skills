# 贡献指南

欢迎贡献新的蒸馏 skill 或改进现有内容。

## 提交新 Skill

### 方式一：提 Issue 请求蒸馏

最简单的方式。提 Issue，说明：
- 人物姓名
- 为什么值得蒸馏（有哪些公开资料：著作/访谈/演讲/论文）
- 适合用在什么场景

我会评估后安排蒸馏。

### 方式二：自己蒸馏后提交 PR

1. Fork 本 repo
2. 使用[蒸馏工具](SKILL.md)或手动撰写 skill 文件
3. 文件命名：`skills/[英文小写名字].md`（如 `skills/paul-graham.md`）
4. 确保满足以下质量标准后提交 PR

## Skill 质量标准

提交的 skill 必须满足：

| 维度 | 要求 |
|------|------|
| **直接引语** | ≥ 5 条，注明来源（书名/演讲/访谈） |
| **心智模型** | 3-7 个，每个有名称 + 支撑引语 + 局限说明 |
| **决策启发式** | 5-10 条，具体到「面对 X 情况，做 Y 判断」 |
| **来源覆盖** | ≥ 3 种来源类型（如博客 + 访谈 + 著作） |
| **诚实边界** | 明确标注调研截止时间、盲区、推断说明 |
| **示例对话** | ≥ 1 个，体现心智模型 + 表达DNA |

## 文件格式

```markdown
---
name: [小写英文]-perspective
description: |
  [人名]的思维框架，基于[来源概述]蒸馏。
  触发词：「[中文名]」「[英文名]」「用[人名]的视角」
type: perspective
调研时间: YYYY-MM-DD
---

# [人名] 思维操作系统

> 蒸馏自：[来源列表]
> 调研截止：YYYY-MM-DD

## 使用说明
...（完整结构见现有 skill 文件）
```

完整结构参考 [skills/xiangshuai.md](skills/xiangshuai.md)。

## 更新现有 Skill

如果发现 skill 内容有误或严重过时：

1. 直接提 PR，说明修改内容和依据
2. 或提 Issue 描述问题，由维护者更新

**注意**：对表达风格、引语的轻微主观分歧不构成更新理由；有明确来源支持的事实性错误才应修正。

## 不接受的内容

- 以侵犯隐私、骚扰或冒充为目的
- 针对非公众人物（普通个人）
- 内容捏造、无来源支撑
- 重复现有 skill 且无实质改进
