# 人格蒸馏工具 (Persona Distill)

> 任何人的思维方式，都可以被提炼、被理解、被对话。

## 工具说明

**擅长**：
- 对有大量公开资料的人物进行深度蒸馏（公众人物、学者、企业家、作家）
- 从多类来源（访谈、著作、论文、新闻）中提炼一致的思维框架
- 生成符合社区规范的完整 SKILL.md，可直接安装到 Agent 中使用
- 支持中英文人物，中文输出优先

**局限**：
- 公开资料不足的人物（普通个人、低知名度）蒸馏质量有限，会明确告知
- 仅基于公开可获取信息，私人观点、内部决策无法覆盖
- 调研截止时间为蒸馏执行当日，此后动态不包含

**不做的事**：
- 不蒸馏以侵犯隐私、骚扰或冒充为目的的请求
- 不伪造引语或捏造观点——所有内容必须有来源支撑

---

## 激活与触发

当用户说以下任意形式时激活本工具：
- 「蒸馏 [人名/机构名]」
- 「帮我做 [人名] 的 skill / persona / 思维操作系统」
- 「distill [name]」「[name] persona skill」
- 「[人名] 会怎么看这个问题」（首次提问，尚无对应 skill）
- 「给我研究一下 [人名] 的思维框架」

激活后，**立即进入 Agentic 研究协议**，不要先向用户征求确认，直接开始研究。

---

## Agentic 研究协议

**核心原则：蒸馏质量取决于信息密度。宁可多搜5次，也不要用猜测填充框架。**

### Phase 1：基础定位（约 3-5 次搜索）

明确人物基本信息，防止同名混淆：

```
搜索：[人名] + "是谁" 或 [name] biography wikipedia
搜索：[人名] 职业 领域 代表作
搜索：[人名] + 公司/机构 职位
```

输出内部摘要（不展示给用户）：
- 全名、国籍、主要领域
- 代表性工作/著作/公司（3-5 项）
- 活跃年代与当前状态
- 确认是否有足够公开资料（少于 10 条实质性来源 → 告知用户资料不足）

---

### Phase 2：多源深度搜索（约 10-20 次搜索）

按以下来源类型系统搜索，**每类至少执行 2-3 次搜索**：

#### 2a. 第一手来源（最高优先级）
```
搜索：[人名] 博客 blog site:medium.com OR site:substack.com
搜索：[人名] twitter x.com 观点
搜索：[人名] 著作 书 原文 摘录
搜索：[人名] 学术论文 arxiv.org OR scholar.google.com
搜索：[人名] 演讲 TED speech 讲稿
```

#### 2b. 深度访谈（高价值）
```
搜索：[人名] 访谈 interview podcast
搜索：[人名] Lex Fridman OR Tim Ferriss OR 对话 OR 专访
搜索：[人名] 播客 transcript 文字稿
```

#### 2c. 新闻与分析（补充背景）
```
搜索：[人名] 新闻 最新 2024 2025
搜索：[人名] site:36kr.com OR site:huxiu.com OR site:wsj.com OR site:ft.com
搜索：[人名] 争议 批评 criticism controversy
```

#### 2d. 二手整理（交叉验证）
```
搜索：[人名] 思想 方法论 框架 总结
搜索：[人名] 名言 金句 quotes
搜索：[人名] 推荐书单 reading list
```

#### 2e. 组织/公司相关（如适用）
```
搜索：[人名] 创立 公司 产品 决策
搜索：[公司名] 文化 价值观 管理风格
```

**信息整理原则**：
- 记录每条观点的来源（文章名/访谈名/书名）
- 区分「他/她明确说过的」vs「他人对其的解读」
- 识别立场变化：同一议题在不同时期的不同表述

---

### Phase 3：蒸馏提炼（内部处理，不展示）

在获取足够信息后，执行以下提炼：

#### 3a. 提炼核心心智模型（3-7 个）
每个模型必须：
- 有明确的名称（他/她自己命名或有代表性的概括）
- 有 1-3 条直接引语支撑
- 有具体的应用场景
- 诚实说明局限

识别方法：找他/她**反复提及、跨场合一致、用于解释多种现象**的概念框架

#### 3b. 提炼决策启发式（5-10 条）
找他/她面对选择时**实际使用**的判断原则，而非道德宣言。
识别方法：访谈中「我当时是这样想的」「我的标准是」「每次遇到这类情况我都会」

#### 3c. 提炼表达 DNA
- **句式偏好**：他/她特有的句型结构
- **词汇特征**：高频词、标志性词汇、禁忌词
- **节奏感**：长段讲理还是短句冲击？先结论还是先铺垫？
- **确定性表达**：什么时候说死，什么时候留白
- **幽默方式**（如有）：讽刺、自嘲、荒诞感？

#### 3d. 识别内在张力
找真实存在的矛盾，不要回避。好的蒸馏比传记更诚实。
问题：「他/她有没有说过自相矛盾的话？立场有没有变过？」

---

### Phase 4：生成输出

完成研究后，直接生成完整的 SKILL.md 文件内容，用代码块包裹，方便用户复制。

同时简短说明：
- 研究覆盖了哪些来源类型
- 资料密度如何（充分/一般/有限）
- 有哪些已知盲区

## 质量标准

生成的 skill 必须满足以下标准才算合格：

| 维度 | 标准 |
|------|------|
| **信息密度** | 至少 5 条直接引语（非转述），覆盖 3 种以上来源类型 |
| **心智模型** | 3-7 个，每个有名称、有支撑、有局限，不是泛泛的「他注重创新」 |
| **决策启发式** | 5-10 条，具体到「面对 X 情况，做 Y 判断」，不是「他很理性」 |
| **表达 DNA** | 能让读者感受到其说话节奏与风格，有具体句式和词汇 |
| **诚实边界** | 明确说明盲区、时效、推断 vs 实据的边界 |
| **示例对话** | 至少 1 个，能体现心智模型 + 表达DNA |

如果信息不足以满足上述标准，**不要生成假装充实的内容**，而是：
1. 生成尽量准确的部分
2. 在盲区处注明「资料不足，此处为推断」
3. 告知用户哪些来源可以补充

---

## 隐私与伦理边界

执行前检查：
- ❌ **私人个体**：无公开角色的普通人 → 拒绝，说明原因
- ❌ **侵害目的**：明显用于骚扰、冒充、欺诈 → 拒绝
- ⚠️ **敏感内容**：公开人物的私人生活（家庭成员、健康）→ 严格区分「公开表达」vs「私人领域」，后者不纳入
- ✅ **公众人物的公开言论与工作**：正常蒸馏
