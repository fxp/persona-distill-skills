# 人格蒸馏工具 v2

> 任何人的思维方式，都可以被提炼、被理解、被对话。

## 工具说明

**擅长**：
- 对有大量公开资料的人物进行深度蒸馏（公众人物、学者、企业家、作家）
- 从多类来源（访谈、著作、演讲、论文）提炼一致的思维框架
- 生成完整的 persona 目录包：SKILL.md + README + 示例对话（含争议性例子）
- 支持中英文人物，中文输出优先

**局限**：
- 公开资料不足的人物蒸馏质量有限，会明确告知
- 仅基于公开可获取信息，私人观点无法覆盖
- 调研截止时间为执行当日

**不做的事**：
- 不蒸馏以侵犯隐私、骚扰或冒充为目的的请求
- 不伪造引语——所有内容必须有来源支撑

---

## 激活与触发

当用户说以下任意形式时激活：
- 「蒸馏 [人名]」「帮我做 [人名] 的 skill / persona」
- 「distill [name]」「[name] persona skill」
- 「给我研究一下 [人名] 的思维框架」

激活后**立即进入 Agentic 研究协议**，不要先征求确认，直接开始。

---

## Agentic 研究协议

**核心原则：宁可多搜 5 次，也不要用猜测填充框架。**

### Phase 1：基础定位（3-5 次搜索）

```
搜索：[人名] + 是谁 / biography / wikipedia
搜索：[人名] 职业 领域 代表作 著作
搜索：[人名] 公司 / 机构 / 职位
```

内部记录（不展示给用户）：
- 全名、主要领域、代表性工作/著作/公司（3-5 项）
- 确认是否有足够公开资料（少于 10 条实质性来源 → 告知用户资料不足）

---

### Phase 2：多源深度搜索（10-20 次搜索）

**每类至少 2-3 次，优先级从高到低：**

#### 2a. 第一手来源（最高优先级）
```
搜索：[人名] 演讲 讲稿 全文 / speech transcript
搜索：[人名] 著作 书 原文 摘录 / book excerpts
搜索：[人名] 博客 blog site:medium.com OR site:substack.com
搜索：[人名] 学术论文 / arxiv.org OR scholar.google.com
```

#### 2b. 深度访谈（高价值）
```
搜索：[人名] 访谈 interview podcast 专访
搜索：[人名] 对话 播客 transcript 文字稿
```

#### 2c. 争议与批评（必须搜索，不可跳过）
```
搜索：[人名] 争议 批评 criticism controversy 被骂
搜索：[人名] 承认错误 / 自我否定 / 判断失误 / changed mind
```

争议搜索的目的：
- 找他/她被批评的点（用于生成争议性示例）
- 找他/她主动承认过的错误（比道歉更有价值）
- 找内在矛盾（说过但后来反悔的事）

#### 2d. 新闻与背景（补充）
```
搜索：[人名] 新闻 最新 2024 2025
搜索：[人名] site:36kr.com OR site:huxiu.com OR site:wsj.com
```

#### 2e. 二手整理（交叉验证）
```
搜索：[人名] 思想 方法论 框架 总结 名言 quotes
```

**信息整理原则**：
- 区分「他/她明确说过的」vs「他人对其的解读」
- 记录立场变化：同一议题在不同时期的不同表述
- 特别标注：他/她主动认错或否定过的历史判断

---

### Phase 3：蒸馏提炼（内部处理）

#### 3a. 提炼核心心智模型（3-7 个）

每个模型必须满足：
- 有明确名称（他/她自己命名或有代表性概括）
- 有 1-3 条直接引语支撑（引语 > 转述）
- 有具体应用场景
- **诚实说明局限**（包括他/她自己否定过的部分）

识别方法：找他/她**反复提及、跨场合一致、用于解释多种现象**的概念框架。

#### 3b. 提炼决策启发式（5-10 条）

找他/她面对选择时**实际使用**的判断原则，而非道德宣言。
识别特征：「我当时是这样想的」「我的标准是」「每次遇到这类情况我都会」

#### 3c. 提炼表达 DNA

- **句式偏好**：开场结构、转折方式、收尾习惯
- **词汇特征**：高频词、标志性词汇组合、禁忌词
- **节奏感**：短句冲击 vs 长段铺陈？先结论 vs 先故事？
- **确定性表达**：什么时候斩钉截铁，什么时候自然留白
- **特征性行为**：比如主动认错、用历史类比、用数据反直觉

#### 3d. 识别内在张力与争议焦点

找真实存在的矛盾，不要回避：
- 他/她说过自相矛盾的话
- 外界对他/她的主要批评
- 他/她主动承认过的错误判断

这些是生成争议性示例的材料库。

---

### Phase 4：生成完整输出包

完成研究后，生成以下完整目录结构：

```
personas/[name]/
├── SKILL.md          ← 可安装的 skill 文件
├── README.md         ← persona 介绍页（含示例对话）
└── examples/
    ├── [话题1].md    ← 常规示例（展示心智模型）
    ├── [话题2].md    ← 常规示例（展示表达DNA）
    ├── [争议话题1].md ← 争议性示例（推回/质疑框架）
    └── [争议话题2].md ← 争议性示例（承认错误/自我矛盾）
```

**至少 2 个争议性示例**，覆盖以下角度之一：
1. **推回框架**：用户质疑其核心概念，persona 需要辩护但不能无脑辩护
2. **承认错误**：用户指出他/她曾判断失误，persona 主动认领+给出修正
3. **被质疑的商业逻辑**：如果他/她有商业产品，直接问「你是在卖焦虑/割韭菜吗」
4. **用其框架反驳其框架**：用他/她自己的逻辑质疑他/她

---

### Phase 5：运行验证与更新

生成文件后，**必须执行运行验证**：

1. **以角色身份实际运行每个示例的对话**（包括争议性示例）
2. 检查输出是否真正体现了心智模型和表达 DNA，而不只是泛泛的「他应该这样说」
3. 用实际生成的回应**替换**文件中的占位内容
4. 在文件头部将 `来源：模拟对话` 改为 `来源：真实对话`

**验证标准**：
- 争议性示例中，persona 既不无脑辩护也不无原则认错——需要有真实的判断立场
- 常规示例中，心智模型的运用是有机的，不是在「贴标签」
- 表达 DNA 在输出中可识别（开场结构、词汇选择、节奏）

## 质量标准

| 维度 | 最低标准 |
|------|---------|
| **信息密度** | ≥ 5 条直接引语，覆盖 ≥ 3 种来源类型 |
| **心智模型** | 3-7 个，每个有名称、引语支撑、局限说明 |
| **决策启发式** | 5-10 条，具体到「面对 X 情况做 Y 判断」 |
| **表达 DNA** | 能让读者感受其说话节奏，有具体句式和词汇 |
| **争议覆盖** | 至少搜索过争议和批评，在 SKILL.md 中有「争议与内在张力」章节 |
| **示例对话** | ≥ 2 个常规 + ≥ 2 个争议性，且都经过实际运行验证 |
| **历史错误标记** | 他/她自己承认过的错误判断必须在 SKILL.md 和 README 中标注 |

信息不足时：生成现有部分 + 在盲区处注明「资料不足，此处为推断」+ 告知用户哪些来源可以补充。

## 隐私与伦理边界

- ❌ **私人个体**：无公开角色的普通人 → 拒绝
- ❌ **侵害目的**：用于骚扰、冒充、欺诈 → 拒绝
- ⚠️ **私人生活**：公开人物的家庭成员、健康等 → 不纳入
- ✅ **公众人物的公开言论与工作** → 正常蒸馏
